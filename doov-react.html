What is doov-react ?

doov-react is an extension tool for doov-ts. If you are not familiar with doov-ts, you can take a look here to get more information before going further with doov-react.
Business rules are not only read by developers, they are also viewed by business people. To facilitate the comprehension of these business rules, we developed a react component <DoovReact>  for rendering these rules in a user-friendly form.

const USER_B: BooleanFunction = DOOV.boolean(DOOV.field<boolean, Model>('user', 'b'));
const USER_ID: NumberFunction = DOOV.number(DOOV.field<number, Model>('user', 'id'));
const USER_NAME: StringFunction = DOOV.string(DOOV.field<string, Model>('user', 'name'));
const USER_LINK0 = DOOV.string(DOOV.field<string, Model>('user', 'links', 0)); //access to the first element of iterable
const USER_LINK1 = DOOV.string(DOOV.field<string, Model>('user', 'links', 1)); //access to the second element of iterable
const USER_LINKS = DOOV.iterable<string>(DOOV.field<string[], Model>('user', 'links'))); //direct access to iterable

/* Multiple validations rule */
const multipleValidationsRule: ValidationRule = DOOV.validations(
      DOOV.when(USER_B.and(USER_ID.greaterOrEquals(2))).validate(),
      DOOV.when(USER_B.and(USER_NAME.endsWith('st'))).validate())

/* Multiple mappings Rule */
const multipleMappingsRule: MappingRule = DOOV.mappings
      DOOV.map(USER_ID.mapTo(StringFunction, v => 'link of ' + v)).to(USER_LINK0),
      DOOV.map(USER_NAME).to(USER_LINK1))
Read business rules
Above produced rules are readable using the componant <DoovReact>.
The only props needed by this component is 'metadata' which is the readable metadata of the rule.
 
1
2
<DoovReact metadata={multipleMappingsRule.metadata}/>;
<DoovReact metadata={multipleValidationsRule.metadata}/>;
