What is dOOv ?

Business rules are sometimes difficult to manage. We give you a tool named dOOv, for writing business rules with a minimal line of code, using a fluent API, easier to write for developers and to understand for business people.
You can use dOOv for type-safe bean validation and mapping. This page is devoted to dOOv written with Java, for Typescript version, see here

Example Rule
This is an example of how we build a validation rule using domain object oriented validation. 
ValidationRule rule = DslSampleModel
   .when(userBirthdate.ageAt(today()).greaterOrEquals(18)
   .and(accountEmail.length().lesserOrEquals(maxEmailSize))
   .and(accountCountry.eq(Country.FR))
   .and(accountPhoneNumber.startsWith("+33")))
   .validate();
Example Run
Then run the rule with your existing JavaBeans by executing the rule on it 
SampleModel beanModel = new SampleModel();
...
beanModel.setAccount(new Account());
beanModel.getAccount().setPhoneNumber("01 42 90 90 90");
...
Result result = rule.executeOn(beanModel);
Example commodities
With can explore the result as below, and even more. 
System.out.println(rule.readable());
// out : "rule when ((user birthdate age at today > 18
//           and account email length is < configuration max email size)
//           and account country = FR)
//           and account phone number starts with '+33' validate"

System.out.println(result.isTrue());
// out : "false"
System.out.println(result.getFailureCause());
// out : "account phone number starts with '+33'"
